# 노트플러스P 모바일 Google 로그인 설계 v1

작성일: 2026-07-22  
대상 후보: `노트앱_v15.html` + `noteplus-cloud-v3.js`

## 목표

- PC뿐 아니라 Android·iPhone 브라우저와 홈 화면 설치 앱에서도 같은 Google 계정으로 로그인한다.
- 로그인만으로 로컬 노트나 첨부를 전송하지 않는다. 사용자가 백업 또는 복원을 명시적으로 눌러야 데이터가 이동한다.
- 로그인 실패·취소·오프라인·브라우저 저장소 차단에도 현재 화면의 입력과 IndexedDB 데이터가 사라지거나 앱이 멈추지 않는다.

## 인증 방식

1. GitHub Pages 공개본의 기본값은 `signInWithPopup`이다.
2. Firebase 공식 문서는 모바일에서 redirect를 권장하지만, Firebase 외부 호스팅은 최신 브라우저의 제3자 저장소 차단을 피하기 위한 프록시·동일 도메인 인증 도우미가 필요하다.
3. 운영자가 Firebase Hosting 또는 공식 권장 프록시를 구성한 경우에만 공개 설정의 `mobileAuthMode: "redirect"`로 전환한다.
4. redirect를 선택한 경우 앱 시작 시 `getRedirectResult`를 처리하고, 성공·실패·무결과를 사용자에게 구분해 고지한다.

## 데이터 안전

- 인증 상태는 Firebase `browserLocalPersistence`를 사용하되 실패 시 앱 전체를 중단하지 않고 Google 기능만 비활성화한다.
- 인증 중 앱 상태·첨부 저장소를 읽거나 쓰지 않는다.
- 백업과 복원은 v14의 첨부 SHA·UID 바인딩·마지막 pointer 커밋·복원 전 자동 보관·롤백을 그대로 사용한다.
- 계정 변경 감지는 백업·복원 전 과정에서 유지한다.

## UI

- 제목을 `Google 계정 보호 · PC·모바일 베타`로 바꾼다.
- 휴대폰에서는 로그인 방식과 복귀 중 상태를 명확히 표시한다.
- 같은 계정으로 로그인해도 자동 동기화가 아니라 명시적 백업/복원임을 항상 표시한다.
- 360px·390px에서 버튼과 상태 문구가 가로로 넘치지 않아야 한다.

## 검증 순서

1. `new Function` 정적 검사와 ES module 구문 검사
2. 모바일 인증 전략·오류 고지·로그인 무데이터변경 회귀
3. 기존 schema·저장 차단·ENEX·첨부·백업·복원·보안 회귀
4. 360px·390px 모바일 시각 검사와 스크린샷
5. 공개 v15 HTTP·SHA·공개 URL 회귀

## 외부 활성화 관문

- 실제 Google 로그인 성공은 Firebase 프로젝트, Web App 설정, Google 공급자, 승인 도메인이 필요하다.
- 실제 첨부 클라우드 백업은 Cloud Storage·UID Rules와 현재 요금제 조건 확인이 추가로 필요하다.
- 이 외부 설정 전에도 로컬 앱은 완전히 작동하며 Google 버튼은 비활성 상태와 이유를 정직하게 표시한다.
