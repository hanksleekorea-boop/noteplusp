# 노트플러스P Evernote 전체 이전 온보딩 재설계 기획서

기준일: 2026-07-19  
대상 정본: `노트앱_v5.html`  
목표: 처음 보는 비개발자도 설명을 해석하지 않고 화면의 한 행동씩 따라가며 모든 노트와 첨부파일의 ENEX 백업·가져오기를 완료한다.

## 1. 반드시 정직하게 알릴 사실

- Evernote 공식 내보내기는 Windows/Mac 데스크톱 앱에서만 가능하다.
- 공식 기능에는 계정 전체를 한 번에 내보내는 버튼이 없다. 개별 선택은 최대 100개, 노트북 전체는 한 번에 1개다.
- `한 번에 가져오기`는 모든 노트북 ENEX를 한 폴더에 모은 뒤 노트플러스P에서 폴더를 한 번 선택한다는 뜻이다.
- 핵심 이전 대상은 노트 제목·본문·태그·날짜·출처 주소와 이미지·PDF·녹음 첨부다.
- 작업·알림, 노트 기록, 공유 권한, 스택·바로가기, 계정 설정, 암호화 블록은 완전 이전을 보장하지 않는다.
- Evernote 원본과 ENEX 백업 폴더는 결과 검증이 끝날 때까지 삭제하지 않는다.

## 2. 현재 화면의 실측 문제

- 6단계·경고·고급 도구·완료 기준이 한 긴 모달에 몰려 인지 부담이 크다.
- 모바일에서도 긴 절차를 보여주지만 실제 내보내기는 PC가 필요하다.
- 공개 Windows 화면 예시는 실제 화면이지만 설치 버전에 따라 메뉴 위치가 다를 수 있다.
- 노트북을 빠뜨리지 않았는지 사용자가 기억에 의존한다.
- `모든 파일` 표현이 작업·기록·공유 권한까지 포함하는 것으로 오해될 수 있다.

## 3. 새 5단계 사용자 흐름

1. **범위·PC 확인**: PC 필요, 공식 제한, 포함/별도 확인 데이터를 먼저 한 화면에서 고지한다.
2. **PC 준비**: Evernote 동기화 완료와 `Evernote 전체 백업` 폴더 생성을 한 행동으로 안내한다.
3. **실제 화면대로 ENEX 저장**: `Notebooks → … → Export notebook…`, `ENEX + 모든 속성 + 300MB` 두 화면만 보여준다.
4. **노트북 반복 체크**: 사용자가 노트북 이름을 붙여 넣어 체크리스트를 만들고 저장할 때마다 완료 표시한다. 체크리스트는 현재 탭에서만 유지된다고 정직하게 고지한다.
5. **폴더 한 번 선택**: 기대 노트·노트북 수를 입력하고 폴더를 선택한다. 기존 저장공간 점검·미리보기·중복·첨부 진단·되돌리기·결과 보고서 흐름으로 연결한다.

각 단계는 이전/다음 버튼과 직접 단계 이동을 지원하고 `aria-current="step"`, 숨김 패널, 키보드 포커스 이동을 제공한다.

## 4. 화면 원칙

- 한 화면에는 한 목적과 하나의 주 행동만 둔다.
- 중요한 메뉴명은 한국어/영어를 함께 표시한다.
- 실제 화면이 다를 때 사용할 우클릭 대체 경로를 이미지 바로 아래에 둔다.
- 모바일에서는 첫 단계에 `PC에서 계속하세요`를 가장 크게 표시한다.
- 자동 백업 도우미는 공식 기본 경로와 분리한 `고급 선택`으로만 제공한다.
- 외부 이미지 URL을 사용하지 않고 기존 로컬 화면 자산만 사용한다.
- 사용자 입력은 `textContent`로만 렌더하며 HTML로 삽입하지 않는다.

## 5. 데이터·보안·저장 원칙

- schema 5, `notes_app_v1`, IndexedDB 상태/Blob 분리 구조는 변경하지 않는다.
- ENEX/JSON/붙여넣기 HTML은 기존 화이트리스트 새니타이저를 그대로 통과한다.
- 체크리스트는 가져오기 데이터가 아니며 현재 탭의 메모리·DOM에만 둔다. 탭을 닫으면 초기화될 수 있음을 표시한다.
- 폴더 선택 전에는 `state`, IndexedDB, localStorage, 기존 첨부를 변경하지 않는다.
- 폴더 선택 뒤 기존 350MB 단일 파일, 500MB 자동 묶음, 저장공간 사전 점검을 유지한다.

## 6. 완료 조건

- 5개 단계 중 한 단계만 화면에 보인다.
- PC 필요와 `Evernote에서는 노트북별 / 노트플러스에서는 폴더 한 번`이 첫 화면에 보인다.
- 포함/별도 확인 데이터가 첫 화면에 구분된다.
- 두 실제 화면이 로컬 자산으로 로드되고 외부 `img src`는 0개다.
- 노트북 이름 붙여 넣기, 중복 제거, 체크 진행률, 삭제가 동작한다.
- 체크리스트 수가 기대 노트북 수 기본값으로 연결된다.
- 폴더·다중 파일 선택기 연결, Escape, 포커스 트랩, 다크모드, 모바일 레이아웃이 동작한다.
- 기존 schema 4→5 이전, ENML/JSON 보안, 첨부 Blob, 저장 실패 폴백, 중단·재시도·중복 회귀가 모두 통과한다.

## 7. 검증·배포 순서

1. `new Function` 정적 구문검사
2. 새 단계형 튜토리얼·체크리스트 자동검사
3. 기존 ENEX 기능·보안·마이그레이션 전체 회귀
4. 데스크톱·모바일·다크모드 스크린샷 직접 확인
5. 서비스워커 캐시 갱신
6. 커밋·푸시 후 공개 파일 SHA-256과 화면 반영 확인

## 8. 되돌리기

- 앱은 단일 HTML이므로 이전 커밋의 튜토리얼 CSS·HTML·이벤트 블록과 서비스워커 캐시 버전을 복원하면 된다.
- 저장 schema와 사용자 노트 데이터는 이번 변경 대상이 아니므로 데이터 롤백은 필요하지 않다.
