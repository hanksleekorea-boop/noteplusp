# 노트플러스P 제한 알파 · 비개발자 10분 파일럿 카드 v1

공개 앱: https://hanksleekorea-boop.github.io/noteplusp/

현재 버전: v11

파일럿 샘플 ENEX: https://hanksleekorea-boop.github.io/noteplusp/pilot-assets/%EB%85%B8%ED%8A%B8%ED%94%8C%EB%9F%AC%EC%8A%A4P_%ED%8C%8C%EC%9D%BC%EB%9F%BF_%EC%83%98%ED%94%8C.enex

목적: 설명을 듣지 않은 사용자가 공개 링크에서 노트를 저장하고, Evernote 전체 이전 방법을 이해하고, ENEX와 PDF 첨부를 가져올 수 있는지 확인한다.

예상 시간: 10분

## 참가자에게 먼저 보여줄 내용

이 테스트는 사용자를 시험하는 것이 아니라 앱이 충분히 쉬운지 시험합니다. 막히면 그대로 멈추고 생각나는 말을 소리 내어 말해주세요. 정답을 맞힐 필요가 없습니다.

개인 Evernote 파일은 사용하지 않습니다. 위의 파일럿 샘플만 사용합니다. 가능하면 Edge 또는 Chrome의 새 InPrivate/시크릿 창에서 시작하세요.

### 1. 앱 열기 · 1분

공개 앱 링크를 여세요. 첫 화면만 보고 이 앱으로 무엇을 할 수 있을 것 같은지 한 문장으로 말해주세요.

성공 모습: 앱이 오류 없이 열리고 노트 목록·새 노트·Evernote 이전 시작점 중 하나 이상을 알아봅니다.

### 2. 노트 만들고 다시 확인하기 · 2분

새 노트를 만들어 다음 내용을 입력하세요.

- 제목: `파일럿 저장 확인`
- 본문: `새로고침 뒤에도 이 문장이 보여야 합니다.`

저장됐다고 생각되면 페이지를 새로고침하고 같은 노트를 다시 찾으세요.

성공 모습: 제목과 본문이 그대로 보입니다. 저장이 제한된 환경이면 앱이 그 사실을 분명히 알려줍니다.

### 3. Evernote 전체 이전 방법 찾기 · 2분

`전체 이전 시작` 또는 `Evernote 전체 이전 도우미`를 찾아 여세요. 안내를 보고 다음 세 질문에 소리 내어 답하세요.

1. Evernote 내보내기는 웹·휴대폰과 PC 데스크톱 앱 중 어디에서 하나요?
2. 계정 전체를 한 번에 내보내나요, 노트북별로 내보내나요?
3. 노트플러스P에서는 여러 ENEX가 모인 폴더를 몇 번 선택하나요?

성공 답:

1. Windows 또는 Mac Evernote 데스크톱 앱
2. 노트북별로 각각 ENEX 저장
3. ENEX 폴더를 한 번 선택

### 4. 샘플 ENEX와 PDF 가져오기 · 3분

1. 카드 상단의 `파일럿 샘플 ENEX`를 내려받으세요.
2. 앱에서 `준비된 ENEX 모두 선택`을 누르세요.
3. 받은 `.enex` 파일을 선택하세요.
4. 미리보기에서 파일 1개·노트 1개·첨부 1개인지 확인하세요.
5. 가져오기를 완료하고 `노트플러스P 파일럿 샘플 노트`를 여세요.
6. `노트플러스P_파일럿_확인.pdf` 첨부가 보이는지 확인하세요.

성공 모습: 샘플 노트가 정확히 한 번 생기고 PDF 첨부가 표시됩니다.

### 5. 다시 쓰는 방법 찾기 · 1분

앱을 다음에 다시 열기 위한 `홈 화면에 추가` 기능을 찾아보세요. 실제 설치는 하지 않아도 됩니다.

성공 모습: 휴대폰에서는 `정리`, PC에서는 왼쪽 메뉴에서 해당 기능을 찾습니다.

### 6. 마지막 한마디 · 1분

다음 두 질문에 답해주세요.

1. 실제 내 Evernote 자료를 옮길 자신감은 1~5점 중 몇 점인가요?
2. 가장 헷갈렸거나 불안했던 한 가지는 무엇인가요?

### 파일럿 1명 성공 기준

다음을 모두 만족하면 이 참가자의 핵심 여정은 통과다.

- 10분 이내 완료
- 앱 진입 성공
- 노트가 새로고침 뒤 유지되거나 저장 제한이 명확히 인지됨
- Evernote 이전 질문 3개 중 3개 정답
- H0 또는 H1만으로 ENEX 노트 1개·PDF 1개 가져오기
- 입력 데이터가 사라지거나 앱이 멈추지 않음
- 자신감 3점 이상

### 결함 우선순위

- P0: 앱 진입 불가, 핵심 가져오기 불가, 데이터 손실, 외부 콘텐츠 실행, 완료 여부 오도
- P1: 재접속 저장 실패, 저장 제한 고지 누락, 안내를 따라도 핵심 과업을 끝내지 못함, H2/H3가 필요한 반복 장애
- P2: 문구·간격·미세한 탐색 불편, 알파 핵심 여정을 막지 않는 개선

P0은 즉시 수정한다. P1은 두 번째 환경에서 재현한 뒤 최소 범위로 수정한다. P2는 알파 종료를 막지 않는다.

1. 참가자가 원하면 결과 화면만 개인정보 없이 캡처한다.
