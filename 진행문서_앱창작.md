- 무손실 이전: 기존 `notes_app_v1` schema 1~4를 읽어 IDB에 쓴 뒤 핵심 상태 서명을 다시 읽어 검증하고 완료 메타를 기록. 원본 localStorage는 성공 후에도 자동 삭제·덮어쓰기하지 않음. 재실행 시 중복 노트가 생기지 않음을 자동 검증.
- 실패 안전성: IDB 실패 시 schema 5 전용 보조 localStorage로 노트 상태를 유지하고 첨부 Blob은 세션 메모리에 유지. localStorage까지 막히면 입력·편집·첨부 기능은 메모리에서 계속 작동하며 상단 배너와 저장 상태로 한계를 정직하게 고지. 수동 저장 재시도 제공.
