## 1. 실제 Evernote 데이터 호환성 결과

- 실제 Evernote 11.25.6 ENEX는 노트 1,950개, 첨부 리소스 2,272개, 첨부가 있는 노트 430개다.
- ENEX는 UTF-8·XML 시작/종료·노트 시작/종료·Base64 형식 검사에서 구조 이상이 없었다.
- v11은 표준 ENEX의 해시 없는 `<data encoding="base64">` 첨부를 거부해 2,272개를 제외했다. 원본 손상이 아니라 앱 호환성 P0이었다.
- v12는 유효 MIME·Base64 첨부를 모두 Blob으로 보존한다. 이미지·오디오·PDF만 코드 통제 DOM으로 미리보기/열기를 제공하고, HTML·Office·ZIP·연락처 등은 실행하지 않고 다운로드 전용으로 제공한다.
- 공개 v12에서 실제 파일은 노트 1,950개·첨부 2,272개·첨부 오류 0개로 미리보기에 도달했고, 확인 전 기존 노트·휴지통은 변하지 않았으며 페이지 오류는 0건이었다.
- 개인정보 보호를 위해 실제 파일은 미리보기에서 취소했다. 사용자의 실제 브라우저로 최종 가져오기 확정과 수량 대조는 남아 있다.

## 7. 제한사항과 다음 실행

- 대형 ENEX는 고정 제품 용량 제한 없이 처리하지만, 브라우저 메모리와 실제 IndexedDB 저장공간은 필요하다.
- HTML·Office·ZIP·연락처 같은 비미리보기 형식은 저장하지만 앱 안에서 실행하지 않는다. 첨부 카드의 `파일 저장`을 통해 내려받는다.
- 다음 사용자 행동은 공개 v12에서 같은 ENEX를 다시 선택하고 `노트 1,950개·첨부 2,272개`를 확인한 뒤 `선택한 노트 가져오기`를 누르는 것이다.
