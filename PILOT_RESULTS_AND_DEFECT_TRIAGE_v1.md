# 노트플러스P 제한 알파 파일럿 결과·결함 판정표 v1

실제 참가자 3~5명의 결과만 기록한다. 준비 문서나 자동검사는 참가자 결과로 입력하지 않는다. 이름·이메일·노트 내용은 적지 않고 `P01` 같은 익명 코드만 사용한다.

## 공통 과업

1. 공개 링크 열기
2. 노트 작성 후 새로고침해 저장 확인
3. Evernote 전체 이전 안내에서 다음 행동 설명
4. 제공된 샘플 ENEX와 PDF 가져오기
5. Google 로그인과 `로그인만으로 전송되지 않음` 이해
6. 클라우드 백업 확인·복원 미리보기에서 현재 데이터 무변경 이해

성공은 `통과`, 도움 뒤 성공은 `통과(H1~H3)`, 실패는 `실패`, 실행하지 않은 항목은 `미실행`으로 기록한다.

## 자동 결함 판정

| 관찰 | 등급 | 처리 |
|---|---|---|
| 앱 진입 불가, 노트·첨부 손실, 다른 계정 데이터 노출, 저장 성공 오도 | P0 | 즉시 알파 중단·재현·수정·전체 회귀 |
| 재접속 저장 실패, 백업·복원 핵심 과업 2명 이상 실패, 저장 범위 고지 누락 | P1 | 두 번째 환경 재현 후 최소 수정 |
| 1명만 H2 이상 필요 | P1 후보 | 같은 과업을 다음 참가자에게 우선 재검증 |
| 문구·간격·발견성 불편이지만 과업 성공 | P2 | 알파 종료를 막지 않고 베타 큐로 이동 |

## 집계

- 완료 참가자: `0 / 3~5`
- 설명 없는 핵심 여정 통과: `0 / 3~5`
- P0: `미판정`
- 재현 P1: `미판정`
- P2: `미판정`
- 알파 관문 판정: `실제 결과 입력 전 미통과`

## 종료 규칙

참가자 3명 이상이 완료되고 P0가 0이며, 반복 재현되는 P1을 수정·재검증하고, 남은 P1/P2·데이터 한계·Firebase 비용 상태를 종료 보고에 기록해야 파일럿 관문을 통과시킨다.
