# 노트플러스P 제한 알파 최단 경로 v1

기준일: 2026-07-19

공개 앱: https://hanksleekorea-boop.github.io/noteplusp/

앱 버전: `노트앱_v5.html` · 저장 schema 5 · 서비스워커 `noteplusp-v5-shell-12`

## 1. 현재 진단

### 확인된 사실

- 공개 루트, 앱, 서비스워커, manifest가 HTTP 200이며 로컬 정본과 공개 파일 SHA-256이 일치한다.
- 공개 앱 SHA-256은 `F9DC9C17B415FB8817FE2439A293E51B93075C75C2BDA4A06435D31FD3202CDA`다.
- 백엔드·계정·클라우드 동기화 없이 브라우저 IndexedDB에 상태와 첨부 Blob을 분리 저장한다.
- 노트 생성·편집·검색, 노트북·태그, 휴지통, 백업·복원, ENEX 가져오기, 이미지·PDF·녹음, 첨부 무결성 검사가 구현돼 있다.
- `tests/test_public_alpha_journey_v1.mjs`에서 390×844 모바일 화면의 노트 생성·편집·새로고침 복원이 공개 URL과 실제 Microsoft Edge로 통과했다.
- IndexedDB와 localStorage를 동시에 차단한 공개 URL에서 입력 유지, 무중단, 세션 한정 정직 고지가 통과했다.
- 현재 저장소에는 과거 문서가 언급한 기존 `tests/`와 `artifacts/`가 없었다. 과거 통과 기록은 참고 증거지만 현재 재실행 증거가 아니다. 이번에 제한 알파 핵심 회귀부터 새 버전으로 복구했다.
- 현재 작업 폴더에는 사용자의 실제 Evernote ENEX와 운영 노트가 없다. 서버가 사용자 브라우저 데이터를 수집하지 않으므로 저장소만으로 실제 사용자 데이터도 확인할 수 없다.

### 아직 확인되지 않은 것

- 실제 Android 또는 iPhone에서의 전체 핵심 여정
- 사용자의 실제 한국어 Evernote 전체 ENEX 가져오기와 수량 대조
- Windows PC 전체 백업 도우미의 실제 계정 OAuth·전체 내보내기
- 비개발자 파일럿 3~5명의 과업 성공률과 막힘
- 실제 대규모 계정에서의 처리 시간·저장공간 한계

### 결함과 제한

- P0: 현재 자동 공개 핵심 여정에서 발견되지 않음. 실제 ENEX·실기기·파일럿 범위는 미검증이므로 전체 P0=0 선언은 보류한다.
- P1: 재현된 제품 결함 없음. 실제 모바일·실제 ENEX·파일럿 증거 부재와 기존 회귀 자산 유실은 검증 차단 요소다.
- P2: 공개 `/favicon.ico`가 HTTP 404다. 앱 진입·저장·PWA 핵심 자산에는 영향이 없어 알파 비차단으로 기록한다.
- 데이터 한계: ENEX는 표준 노트·노트북·태그·첨부 중심이며 작업·알림 등 Evernote 비공개/미제공 범위는 완전 이전을 보장하지 않는다.

## 2. 제한 알파 핵심 사용자 여정

1. 사용자가 공개 링크를 휴대폰에서 연다.
2. PC 안내에 따라 Evernote 노트북을 ENEX로 내보낸다.
3. 노트플러스P에서 준비한 ENEX를 한 번에 선택하고 미리보기 수량을 확인한다.
4. 가져온 노트를 휴대폰에서 찾고 열고 수정한다.
5. 앱을 닫았다 다시 열어 수정 내용과 첨부가 남았는지 확인한다.
6. 저장 불가 환경에서는 입력이 사라지지 않고 유지 범위가 명확히 표시되는지 확인한다.

수익화, 제휴, 계정 동기화, 협업, 작업·알림 완전 이전, 대규모 최적화, 고급 편집 기능은 이 여정의 P0/P1을 만들지 않는 한 베타 이후로 미룬다.

## 3. 알파 종료 관문 5개

| 관문 | 현재 상태 | 근거와 남은 일 |
|---|---|---|
| 1. 누구나 여는 공개 릴리스 | 통과 | 공개 HTTP 200, 정본·공개 SHA 일치 |
| 2. 실제 모바일 핵심 여정 | 부분 | Edge 모바일 화면 자동 여정 통과. 실제 Android/iPhone에서 가져온 노트 찾기·수정·재접속 필요 |
| 3. 저장·데이터 안전 | 통과 | 공개 URL IndexedDB 재접속 저장과 완전 차단 시 무손실·무정지·정직 고지 통과 |
| 4. 비개발자 파일럿 3~5명 | 미통과 | 참여자·실제 ENEX 필요 |
| 5. 종료 품질 판정 | 부분 | 현재 자동 범위 P0 없음, P2 favicon 기록. 실제 ENEX·파일럿 뒤 P0=0 및 남은 P1·한계 보고 필요 |

**현재 제한 알파 진척도: 2 / 5 관문 통과, 2개 부분 통과, 1개 미통과. 공식 통과율 40%.**

기존 83%는 기능 구현량 기반의 과거 참고치다. 앞으로 알파 진척도는 위 5개 사용자 검증 관문만으로 보고한다.

## 4. 최단 임계 경로

`공개 릴리스 무결성` → `공개 모바일·저장 회귀` → `실제 ENEX 1계정 이전` → `실제 휴대폰에서 찾기·수정·재접속` → `3~5명 파일럿` → `P0 즉시 수정·P1 최소 수정` → `종료 보고`

- Codex 즉시 수행: 공개 회귀 복구, 합성 ENEX 안전 회귀, 수량 대조 UX, 결함 분류, 파일럿 스크립트·결과 양식, 공개 배포 검증.
- 사용자 또는 참여자 필요: 실제 ENEX 제공 또는 화면 공유, Android/iPhone 실기기, 비개발자 3~5명 참여.
- 현재 API 키·계약·결제는 필요하지 않다.
