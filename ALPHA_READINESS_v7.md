현재 정본: `노트앱_v9.html`

서비스 워커: `noteplusp-v9-shell-1`

이 문서는 공개 v9 키보드·포커스·ARIA P1 수정과 공개 회귀 이후의 상태 정본이다. v6 이하 대시보드는 당시 증거로 보존한다.

## 1. 현재 사실과 판정

- 공개 v8 접근성 감사에서 노트 선택, 노트북·태그 접기, 노트북 추가가 마우스 전용이고 저장·가져오기 상태가 화면낭독기에 전달되지 않는 P1을 재현했다.
- v8은 보존하고 v9에 키보드 Enter·Space 조작, 노트 목록 listbox/option 상태, 노트 본문 textbox 이름, 저장·가져오기 상태 live region, ENEX progressbar 값, 대화상자 초점 순환·복귀, 공통 `:focus-visible` 표시를 추가했다.
- 공개 v9 최신 회귀 11종은 모두 통과했다. 접근성 검사의 최초 공개 실행은 서비스 워커 전환 중 일시적 스크립트 가져오기 콘솔 오류로 최종 판정만 실패했으나, 독립 재실행에서 기능·콘솔 모두 통과했다.
- ENEX 취소·재시도 검사에서 노트 상태와 첨부 Blob 저장의 비동기 완료 시점이 달라 첫 실행이 실패했다. 제품 데이터 손실은 없었고 검사에 Blob 저장 완료 대기를 추가한 뒤 통과했다.
- 공개 v9는 HTTP 200, 240,752바이트이며 로컬·공개 SHA-256이 일치한다.
- 실제 Samsung Android는 v8에서 통과했지만 v9 공개 뒤에는 무선 ADB 장치가 검색되지 않았다. 현재 공개 버전의 실기기 재확인이 필요하므로 모바일 관문은 임시로 부분 통과로 내린다.
- 현재 검증 범위 미해결 P0·P1은 0건이다. `/favicon.ico` 404는 P2다.
- 실제 한국어 사용자 ENEX 1세트, v9 실제 Android/iPhone, 비개발자 파일럿 3~5명은 외부 의존성이다.

## 2. 제한 알파 관문

| 관문 | 상태 | 근거와 남은 일 |
|---|---|---|
| 1. 누구나 여는 공개 릴리스 | 통과 | 공개 v9 HTTP 200, 로컬·공개 SHA 일치 |
| 2. 실제 모바일 핵심 여정 | 부분 | v8 Samsung 통과·v9 390px 자동 회귀 통과. 현재 v9 실기기 재확인 필요 |
| 3. 저장·데이터 안전 | 통과 | IDB·보조 저장·전체 차단·schema 4→5·외부 HTML·백업 복원 통과 |
| 4. 비개발자 파일럿 3~5명 | 미통과 | 실제 참여자 필요 |
| 5. 종료 품질 판정 | 부분 | 현재 범위 P0/P1=0. 실제 ENEX·v9 실기기·파일럿 종료 보고 필요 |

**현재 공식 진척도: 통과 2/5, 부분 통과 2, 미통과 1. 공식 통과율 40%.**

## 7. 증거와 제한

- 접근성 검사: `tests/test_public_accessibility_v1.mjs`
- 시각 증거: `artifacts/public_accessibility_v1.png`
- v8 고정 회귀의 기본값은 그대로 유지하고 `NOTEPLUS_APP_FILE`, `NOTEPLUS_EXPECTED_SHA`, `NOTEPLUS_PUBLIC_URL` 환경변수로 새 버전을 검사할 수 있게 했다.
- 접근성 스크린샷의 파란 외곽선은 실제 키보드 초점 표시다.
- 휴대폰은 현재 `adb devices`와 mDNS 검색에서 발견되지 않았다. 사용자가 무선 디버깅을 다시 켜면 v9 실기기 검사를 최우선으로 실행한다.
